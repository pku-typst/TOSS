# Community Distribution Boundary

Everything under this directory must be safe for a public GitHub repository.
Use neutral names, public assets, synthetic examples, and generally available
services only. Never add downstream branding, proprietary packages or fonts,
private hosts, credentials, personal data, or deployment-specific configuration.

Open-source dependencies authored by a vendor are allowed when their licenses
permit redistribution; that does not make vendor-specific product identity or
private assets part of the Community distribution.

The community distribution is the fallback used when `TOSS_CONFIG` is absent,
so keep it self-contained and runnable without private infrastructure.
