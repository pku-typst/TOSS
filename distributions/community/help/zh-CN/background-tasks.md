## 发起后台任务

登录用户可以通过应用页眉中的 **任务中心** 查看持久任务。在 LaTeX 项目的预览工具栏中，如果部署启用了对应能力，可以使用 **在后台构建 PDF** 请求原生构建。

提交构建时，系统会捕获一份服务端已经接受的不可变项目快照，其中包括入口文件、项目设置、文本文件和附件。如果某次修改或上传必须进入本次构建，请先等待它同步完成。提交后的编辑可以照常继续，但只会进入之后的新构建。

操作按钮会反映部署容量：

- **在后台构建 PDF**：立即提交到持久任务队列。
- **排队后台构建；Worker 暂时离线**：允许在容量上限内排队，等待兼容 Worker 恢复。
- **后台 PDF 构建当前不可用**：当前部署没有为此操作配置兼容的处理器。

## 浏览器预览与原生处理

Typst 和 LaTeX 的实时预览仍然在浏览器中编译。后台构建是使用隔离原生工具链的独立显式操作；它不会成为首选预览编译器，也不会作为自动后备。即使后台容量不可用，浏览器预览和本地 PDF 下载仍可正常使用。

浏览器与原生工具链使用彼此独立的宏包、字体和沙箱契约，因此生成的 PDF 可能不同。后台任务会记录实际生成结果的处理器契约。

## 查看、取消与恢复

任务中心会显示当前登录账号的进行中任务和近期任务，包括来源项目、当前阶段、失败详情和更新时间。关闭任务抽屉、切换页面或关闭浏览器都不会取消任务。

任务处于准备、排队或执行阶段时可以请求取消。正在执行的 Worker 需要安全停止，因此取消可能需要短暂等待；结果进入收尾阶段后不能再取消。Worker 临时中断时系统可能自动重试；如果任务最终因编译错误失败，请修复项目后重新提交一次构建。

## 下载与保留结果

任务成功后，可以在任务中心通过产物按钮下载结果。每次显示任务或下载产物时，系统都会重新检查当前账号对来源项目的访问权限。

任务和产物遵循部署方设置的保留策略，并不是永久发布归档。请及时下载重要结果，并继续通过日常备份或 Git 工作流保留源码。

## 发送给 Worker 的数据

与浏览器实时预览不同，后台构建会通过服务端把捕获的项目快照发送给经过认证的处理 Worker。Worker 只会获得本次任务所需的、限定范围且会过期的输入输出权限；它不会获得你的应用会话、数据库凭据、对象存储凭据或外部 Git 授权。
