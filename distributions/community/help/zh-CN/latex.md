## 浏览器 LaTeX 预览

Community 部署可以选择启用 LaTeX 项目。实时编译在持久浏览器 Worker 中运行，并使用项目选择的 pdfTeX 或 XeTeX 引擎。所需 TeX Live 文件会通过服务端按需获取并缓存，因此首次使用新宏包时通常会更慢。

依赖现代字体选择时建议使用 XeTeX；传统宏包和经典工作流可使用 pdfTeX。

## 原生后台 PDF 构建

对于登录用户，部署提供兼容处理容量时，预览工具栏还会显示 **在后台构建 PDF**。这个显式操作会捕获不可变项目快照，在隔离 Worker 中通过原生 TeX Live 2026 和 `latexmk` 构建，并将结果发布到 **任务中心**。切换页面或关闭浏览器后，任务仍会继续。

如果 Worker 暂时离线，可以在容量上限内排队构建；如果没有配置兼容 Worker，只有后台操作不可用，浏览器预览和本地 PDF 下载仍可正常使用。登录后可在帮助中的 **后台任务与任务中心** 查看快照、取消、访问控制与保留行为。

## 兼容性与输出

浏览器运行时覆盖常见文档工作流，但并不是完整的桌面 TeX Live 环境。它不支持 shell escape 或宿主机程序。原生 Worker 提供范围更广且版本固定的 TeX Live 环境及受信任辅助程序，但仍运行在沙箱中，不提供无限制的 shell 或文件系统访问。

浏览器与原生构建使用彼此独立的运行时、宏包、字体和沙箱契约，因此生成的 PDF 可能不同。对于正式发布的关键输出，可以在符合工作流要求时使用原生后台结果，并继续完成组织要求的其他验证。
