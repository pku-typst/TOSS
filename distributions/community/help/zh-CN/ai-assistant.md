## 添加自己的 AI 连接

如果部署启用了可选的助手功能，可从 Workspace 工具栏打开 **助手** 并添加连接。TOSS 不会替你选择 Provider。需要填写：

- 仅在本地显示的连接名称；
- 一种受支持的 API 协议：OpenAI 兼容 Chat Completions、OpenAI Responses 或 Anthropic Messages；
- Provider 或本地服务的 Base URL；
- 该端点能够识别的准确模型 ID；
- 模型是否会返回 reasoning 内容；
- 模型或 Provider 专用请求参数的准确 JSON 对象；
- 该模型的上下文窗口 token 数；
- 该模型的最大输出 token 数。

端点必须使用 HTTPS；只有 `localhost` 和 `127.0.0.1` 可以使用 HTTP。包含凭据、查询参数或片段的 URL，以及与 TOSS 应用同源的 URL，都会被拒绝。

登录账号的这些非秘密连接元数据（包括 reasoning 能力声明、Provider 请求参数和两个 token 上限）会保存在按账号隔离的浏览器存储中。匿名会话可以在当前页面会话中使用连接，但不会保存连接元数据。

## 安全输入凭据

保存非秘密配置后，隔离 Runtime 会显示已绑定的准确目标、协议和模型。请在其中输入 API Key 或短期 Token；无需认证的本地端点可以留空。

凭据只保存在这个不透明源沙箱 Runtime 的内存中。TOSS Core、宿主应用、项目、Local Storage 和 Session Storage 都不会收到凭据。刷新页面、替换 Runtime、切换连接、退出登录或更换账号都会清除内存中的凭据。

Runtime 只允许访问所配置 Base URL 之下的请求，省略浏览器凭据、拒绝重定向，并且只会在接受凭据前加载所选协议实现。

## Provider 与浏览器要求

浏览器会从不透明源直接向模型端点发送请求，因此端点必须接受 `Origin` 为 `null` 的 CORS 请求。本地服务还可能触发浏览器的 Local Network Access 控制。TOSS 不会把失败请求静默改走后端代理。

请求失败时，请检查端点路径、协议、模型 ID、凭据、Provider 请求参数、端点 CORS 策略以及浏览器对本地服务的权限。某个桌面客户端可以成功连接，并不代表浏览器 CORS 已经启用。

## 当前项目访问能力

每次发送消息时，Workspace 都会向助手提供一份小型的当前状态摘要：项目和文档身份、实时或历史版本模式、读写权限、文件数量、同步状态、编译状态与诊断数量，以及是否有修改正在等待审阅。这能让助手了解当前情境，但不会包含源码、完整文件列表或原始诊断。需要准确或更新的数据时，助手仍必须调用下列有边界的工具。

对于 Typst 项目，助手还可以使用本地的 `query_typst_docs` 工具。它会查询随
Runtime 打包的 Typst 0.15.0 API 索引，以及少量经过实际编译验证的用法
recipe，覆盖文档 metadata、content 与 string、set/show 规则、数组和 import
等容易产生“看似合理但无法编译”代码的主题。Typst Runtime 会在隔离启动期间
加载并校验这些参考资料，之后只在私有内存中查询；资料不会注入每一轮上下文，
也不会访问 `typst.app` 或任何后端。模型侧查询和返回的参考文本统一使用英文，
但助手仍会使用用户的语言回答。文档查询只是指导，候选编译和人工审阅仍然是
修改能否成立的权威判断。

助手可以在 `pi-agent-core` 多轮循环中使用三种有边界的只读 Workspace 工具：

- 列出项目目录、文本文档和资源文件；
- 按范围读取一个项目文本文档；
- 在项目文本文档中搜索字面文本。

读取始终针对当前 Workspace 视图。当前活动文档取自最新的协作/编辑器投影；历史版本模式则读取所选的不可变版本。返回给模型的源码采用 `行号 | 代码` 前缀；前缀只是显示信息，并不属于文件内容。切换 Workspace generation 或所选历史版本会替换隔离 Runtime，旧工具调用不会返回到新视图；当前项目所选的本地对话仍会保留，但新 Runtime 需要重新输入凭据。

在可写的实时项目中，助手还可以为当前活动文本文档提交 `apply_patch`。它必须使用读取工具返回的准确快照，并提供一个带上下文的单文件 unified-diff 提案。路径、旧文件起点、上下文和删除行必须与该快照精确匹配；Workspace 会根据通过验证的 hunk 正文推导冗余的行数与新文件坐标，这不会启用模糊匹配或自动 rebase。进入审阅前，Workspace 会构造一个不发布的候选 World，并在懒启动的独立浏览器 worker 中编译。候选版本编译失败时不会进入接受界面；有边界的诊断会返回 Agent，让它继续修改补丁。编译通过后，中央编辑区才显示规范化后的准确 diff、编译通过状态以及 **拒绝** 和 **接受修改** 操作。接受前不会改变项目。接受时，Workspace 会再次检查当前内容、编译 World 和权限，再通过现有协作文档事务写入。任何本地或远程源码变化都会让提案过期。历史版本和只读视图不会暴露此工具。

对于小型整文件重写，或者模型连续无法构造有效 patch 格式时，助手可以改用 `write_file`。它只会在一次 `read_project_file` 从第一行完整读取当前活动文件且结果没有截断后可用；局部读取不能授权整文件替换。助手必须提交完整目标内容，而不是只提交修改行。Workspace 会保留文件统一的 LF/CRLF 约定以及原本已有的末尾换行，自行生成聚焦的 diff，在审阅区明确标记“整文件替换”，并与 `apply_patch` 共用完全相同的隔离编译、时效检查、人工审阅和协作写入流程。首版限制为 400 个逻辑行、65,536 个内容字符；它仍不能创建、删除、重命名文件，也不能写入非活动文件或二进制文件。

候选编译与实时预览状态隔离。Typst 会先进行轻量的纯语法检查，明显的 parser 错误会直接返回而不启动编译器。但纯语法检查会漏掉 import、求值、资源、字体和布局错误，因此通过后仍必须在独立 World scope 中完成完整的仅诊断编译；该过程不导出 PDF/vector 产物，也不触碰预览 renderer。LaTeX 使用独立队列。已下载的 runtime 模块和包仍可复用浏览器缓存，但编译器内存和增量状态不共享；候选 worker 空闲一段时间后会被释放。这些工具仍不能读取二进制资源、查看当前预览诊断、任意触发完整项目编译、创建/删除/重命名文件、原子修改多个文件、调用 Shell、任意访问网络或调用后端。只有已接受的审阅和更新后的编辑器才代表修改确实发生；模型自行声称已经修改并不是证据。

## 对话与活动

可以使用对话记录上方的选择器，在当前项目内新建、切换、重命名或删除对话。切换对话会重置模型上下文，但会保留已经输入当前 Runtime 的凭据。Agent 正在运行时，需要先等待完成或停止运行才能切换。

对于已登录账号，TOSS 会在当前浏览器的 IndexedDB 中保存一份有边界的对话投影，并按账号和项目隔离。保存内容包括用户可见消息、助手可见回答和少量工具活动摘要；不包括凭据、reasoning 文本、系统提示词、原始工具结果、工具返回的源码片段或补丁。最终回答本身可能引用项目内容，因此本地对话历史仍应按项目数据对待。匿名对话只在当前项目页面保持打开期间存在。刷新后可以恢复已登录账号的对话历史，但不能恢复凭据。

回答会流式显示，并支持 Markdown、代码块、表格、基线 KaTeX 数学公式（`$...$`
行内公式，以及起止 `$$` 各自独占一行的块级公式）以及复制最终答案。Typst 或 LaTeX 源码仍应放在
代码块中，而不会作为数学公式渲染。按 Enter 发送，按 Shift+Enter 换行，也可以
用 **停止** 取消当前运行。只有当你仍靠近底部时，对话才会自动跟随新内容；
否则可点击 **跳到最新**。

系统指令以及所有模型可见的工具和参数描述都只维护一份英文定义，不随界面语言变化。界面展示标签仍会本地化；英文系统指令会要求模型使用用户的语言回答。

如果模型返回 reasoning，TOSS 会把它标为模型提供的内容，并与最终答案分开显示。Reasoning 和工具调用会合并到可折叠的 **Agent 活动** 区域：Agent 工作时自动展开，运行结束后自动折叠，除非你手动选择了展开状态。工具卡只显示有边界的说明和状态，不显示原始请求/结果、项目源码或补丁。Reasoning 不保证完整或正确，也永远不能证明某个操作已经发生。

Reasoning 分为两个独立的连接设置。**模型会返回 reasoning 内容** 只向 pi 声明能力，使 reasoning 内容块和多轮历史得到正确处理；它不会自行添加请求字段。**Provider 请求参数（JSON）** 会作为准确对象合并进每一次生成的请求。保留 `{}` 即沿用 Provider 默认；否则请填写具体模型文档要求的结构，例如 `reasoning`、`reasoning_effort`、`thinking`、`chat_template_kwargs` 或 `nvext`。TOSS 不会在这些不兼容的格式之间猜测或转换。

该 JSON 不能替换模型身份、消息、系统指令、工具、tool choice、流式字段、token 上限字段、headers 或凭据。不要把秘密放进去。保存时会拒绝无效 JSON；端点不支持的字段会产生 Provider 错误，而不会自动换成某个猜测值。

## Token 上限与用量

TOSS 把上下文窗口与最大输出作为每个连接的属性，因为用户自定义端点不能被假定会提供可信的模型元数据。请填写端点文档给出的真实上限。每次内部模型调用前，TOSS 都会为配置的最大输出和安全余量预留空间；需要缩减时，先从模型视图移除完整的较旧轮次，再压缩已经完成的较旧工具载荷，同时保留当前请求和最新工具结果。这只影响发送给 Provider 的上下文，不会删除浏览器里可见的本地对话。如果当前请求仍放不下，本轮会以明确的上下文预算错误结束，而不会发送被截断的请求。

对话下方的 token 状态条会显示当前上下文占用、本次用户请求累计输入/输出、模型调用次数，以及是否压缩过较旧上下文。**Provider 已回报**表示所有已完成调用都返回了 token 用量；**部分用量已回报**表示只有部分调用返回；**估算**表示端点没有给出可用 token 数，因此只能显示浏览器的保守上下文估算。缓存和 reasoning 明细可在状态条提示中查看。TOSS 不会把这些计数发送给 Core 或分析服务，也不会据此推算价格。

## 数据流

提示词和对话上下文会从 Runtime 直接发送到所选端点。模型调用 Workspace 工具时，有边界的结果（包括请求读取的项目源码片段）会返回 Runtime，并进入下一次 Provider 请求；经过净化的响应内容、有边界的活动状态和 token 计数再返回宿主界面显示。所选 Provider 自己的计费、保留和隐私条款仍然适用。TOSS Core 不会代理凭据，也不会保存 AI 对话记录或 token 用量；只会在当前浏览器中保留上文所述的有边界本地投影。
