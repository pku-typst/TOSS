## Share a project

Project owners manage access from **Settings → Access**. Use named members for ongoing collaboration. Share links are convenient for a specific review or editing session and can be revoked by the owner.

Read access cannot change project content. Write access can edit project files. Management access controls sharing, templates, and repository integration.

## Work through interruptions

The editor reconnects automatically after a short network interruption. Keep the tab open until the connection indicator returns online. If the service remains unavailable, do not assume unsent edits have reached the server.

## Keep a durable copy

The collaboration service and Git are separate. When the deployment provides Git integration, connect a repository you control and synchronize deliberately. You can also download a project archive at any time.

Check the repository status and last synchronization time before treating Git as current.
